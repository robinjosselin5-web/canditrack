# CandiTrack

## Stack

- React + TypeScript + Vite
- Node.js + Express
- Prisma
- PostgreSQL
- JWT

## Règles

- Respecter l'architecture existante.
- Modifier uniquement les fichiers nécessaires.
- Réutiliser le code existant.
- Éviter les dépendances inutiles.
- Produire un code simple, lisible et fortement typé.
- Ne pas reformater des fichiers sans lien avec la tâche.
- Laisser le projet compilable.

## Conventions

- camelCase : variables et fonctions
- PascalCase : composants et classes
- IExample : interfaces
- UPPER_CASE : constantes
